# Milestone v1.0: 单标的可复现 AI 回测闭环

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 1-6
**Total Plans:** 12

## Overview

v1.0 shipped the full single-symbol, point-in-time, skill-first backtest loop: canonical orchestration, manual snapshot assembly, schema-locked AI decisions, deterministic execution, traceable reporting, and a dedicated validation entry with golden-run regression.

## Phases

### Phase 1: 项目脚手架与运行契约

**Goal**: 把一次回测定义成可重复执行、可落盘、可重放的运行单元。  
**Depends on**: None  
**Plans**: 5 plans

Plans:

- [x] 01-01: 建立 AI 决策契约前的运行边界与总控入口
- [x] 01-02: 固化 run_id 与 status/manifest 语义
- [x] 01-03: 固化 run container 顶层布局
- [x] 01-04: 明确 resume/replay lookup 规则
- [x] 01-05: 补齐 orchestrator 运行序列示例

**Details:** Phase 1 established `backtest-orchestrator` as the single canonical entry, fixed run identity and artifact layout, and defined resume/replay lookup semantics.

### Phase 2: 点时点数据快照管线

**Goal**: 为每个调仓时点构建统一、可缓存、可审计的数据快照。  
**Depends on**: Phase 1  
**Plans**: 3 plans

Plans:

- [x] 02-01: 定义 manual point-in-time snapshot contract
- [x] 02-02: 扩展 run container metadata 与 snapshot validation
- [x] 02-03: 增加 operator-facing snapshot examples

**Details:** Phase 2 defined the manual snapshot boundary, anchored snapshots on `asof_date`, and separated snapshot validation evidence into `meta/snapshot-validation.json`.

### Phase 3: AI 决策契约与策略大脑

**Goal**: 把历史快照稳定地转换成结构化决策对象。  
**Depends on**: Phase 2  
**Plans**: 1 plan

Plans:

- [x] 03-01: 定义 AI decision protocol and schemas

**Details:** Phase 3 locked the JSON Schema contract, fail-closed decision records, and the schema-safe brain-node boundary.

### Phase 4: 回测仿真内核

**Goal**: 用显式交易和资金假设驱动净值变化。  
**Depends on**: Phase 3  
**Plans**: 1 plan

Plans:

- [x] 04-01: 实现执行核心、工件持久化与可审计的净值输出

**Details:** Phase 4 added deterministic next-open execution, lot-aware sizing, explicit cost assumptions, and run-scoped execution persistence.

### Phase 5: 报告与可追溯复盘

**Goal**: 让一次运行既有结果摘要，也有完整证据链。  
**Depends on**: Phase 4  
**Plans**: 1 plan

Plans:

- [x] 05-01: 生成可追溯报告、机器可读索引与完成态收口

**Details:** Phase 5 synthesized the canonical report from execution artifacts only and linked the report back to the run container evidence chain.

### Phase 6: 质量护栏与验证闭环

**Goal**: 确保结果可信、可复跑、可回归。  
**Depends on**: Phase 5  
**Plans**: 1 plan

Plans:

- [x] 06-01: 建立 validate 质量入口、golden run 回归基线与 meta/validation.json 收口

**Details:** Phase 6 added a dedicated validation entry, deterministic run-container checks, and golden-run regression coverage.

## Milestone Summary

**Key Decisions:**

- Use a skill-first orchestrator architecture
- Treat point-in-time snapshot assembly as a first-class boundary
- Require schema-validated AI JSON decisions
- Store run artifacts in a fixed local layout
- Start with a single-symbol closed loop
- Use explicit execution costs and lot-aware sizing for Phase 4
- Derive reports only from execution artifacts for Phase 5
- Add a dedicated validation entry with golden-run regression for Phase 6

**Issues Resolved:**

- Fixed the canonical entry and run container contract
- Fixed point-in-time snapshot auditing and cache reuse rules
- Fixed AI decision schema and fail-closed handling
- Fixed deterministic execution and execution persistence
- Fixed report synthesis and evidence-chain indexing
- Fixed validation determinism and golden-run regression

**Issues Deferred:**

- Multi-symbol portfolio support
- Live trading integration
- Minute-level or tick-level backtesting
- Web UI
- Optimization over prompt/model combinations

**Technical Debt Incurred:**

- None tracked for v1.0 close
