# Pitfalls Research: 龟龟策略

**Date:** 2026-04-23

## 1. 未来函数污染

- **为什么危险**: AI 如果看到了历史时点之后才披露的数据，回测收益会失真。
- **预警信号**: 某些财务字段在过早日期就可用；策略表现异常平滑或过强。
- **预防策略**: 明确“披露日期”和“可见日期”，快照模型只暴露当时可见信息。
- **建议处理阶段**: Phase 2

## 2. 供应商字段漂移与修订

- **为什么危险**: 同一字段在不同源中的命名、精度或口径可能不一致。
- **预警信号**: 同一天同标的在不同数据源中数值明显冲突；重跑结果不稳定。
- **预防策略**: 建立标准快照模型，记录数据源、抓取时间、字段映射版本。
- **建议处理阶段**: Phase 2, Phase 6

## 3. LLM 输出漂移导致解析失败

- **为什么危险**: 自由文本、半结构化 JSON 或字段缺失会直接破坏执行链路。
- **预警信号**: 同一个 prompt 偶发缺字段、动作名称不一致、数值类型变化。
- **预防策略**: 强制使用 JSON Schema / Pydantic 校验，失败时进入显式错误路径。
- **建议处理阶段**: Phase 3

## 4. 提示词与模型版本不可追溯

- **为什么危险**: 回测差异出现时，无法判断是数据、模型还是策略模板导致。
- **预警信号**: 结果“感觉变了”，但没有清楚的 prompt/model/version 记录。
- **预防策略**: 为提示词、策略版本、模型档位建立显式元数据，并随 run 保存。
- **建议处理阶段**: Phase 3, Phase 5

## 5. 执行假设不真实

- **为什么危险**: 不考虑手续费、滑点、仓位限制，会让收益虚高。
- **预警信号**: 高频换手但收益不受影响；现金不足仍然能继续买入。
- **预防策略**: 在账本层引入手续费、滑点、仓位上限和失败订单处理。
- **建议处理阶段**: Phase 4

## 6. 只有结果，没有工件链路

- **为什么危险**: 看见净值曲线却不知道每一步为什么发生，无法定位问题。
- **预警信号**: 报告里只有最终指标，没有快照、决策、成交与版本信息。
- **预防策略**: 以 `run_id` 为中心组织工件目录，保留配置、快照、决策和报告。
- **建议处理阶段**: Phase 1, Phase 5

## 7. 数据权限与成本被低估

- **为什么危险**: Tushare 权限分层和模型调用成本会直接影响系统可持续性。
- **预警信号**: 开发中后期才发现数据接口无法用，或大规模回测成本过高。
- **预防策略**: 从第一天就做 provider fallback 与模型档位切换。
- **建议处理阶段**: Phase 2, Phase 3
