## Integration Review

The canonical flow remains aligned with the skill topology:

`总控 -> 数据 -> 研判 -> 执行 -> 复盘 -> 验证`

Observed handoffs are consistent:

- `backtest-orchestrator` routes work to `data-node`, `brain-node`, `execution-node`, `report-node`, and `validation`
- `brain-node` emits schema-validated decisions only
- `execution-node` consumes validated decisions and writes execution artifacts only
- `report-node` consumes completed execution outputs and writes report artifacts only
- `validation` consumes the completed run container and writes `meta/validation.json` only
- Phase 2 snapshot validation uses `meta/snapshot-validation.json`, so it no longer collides with the final quality-validation record

`execution` no longer pre-creates `report.md`; the canonical report file is owned exclusively by `report-node`.

The validation and reporting contracts explicitly preserve the separation between acquisition, decision, execution, report synthesis, and quality checking.
