<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phase 2 snapshot -> execution engine | Normalized snapshot and price series are trusted only after point-in-time validation; malformed or future-dated data must not progress. |
| Phase 3 decision record -> execution node | Decision payloads must be schema-validated JSON; free-form text or unexpected fields are treated as blocked input. |
| execution engine -> run artifacts | File writes must be deterministic and run-scoped so a replay cannot silently mutate identity or cross-run state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | S/T | `src/guigui_strategy/execution/node.py` input validation | mitigate | Validate the decision record against `docs/contracts/ai-decision-record.schema.json` and reject any payload that is missing required fields or contains unexpected values before the engine runs. |
| T-04-02 | T/R | `src/guigui_strategy/execution/persistence.py` artifact writes | mitigate | Use atomic writes into `{artifact_root}` and include blocked-record reasons plus deterministic metadata so the run can be replayed and audited without guessing which output is authoritative. |
| T-04-03 | D | `src/guigui_strategy/execution/engine.py` price lookup | mitigate | Bound the scan to the Phase 2 price series and fail closed when the next open after `asof_date` is unavailable, instead of looping or fabricating a fill. |
| T-04-04 | I | `execution/rejections.jsonl` and ledger outputs | accept | The artifacts remain inside the local run container; exposing blocked decisions is required for audit and is acceptable because no external sharing path is introduced. |
</threat_model>
